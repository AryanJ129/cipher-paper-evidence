# Scale benchmark — method

Exactly how the scaling numbers in `results/scale_benchmark_2026-06-07.jsonl`
(paper ledger rows Q27–Q30) were produced. The runner itself is **not** published:
it imports the private `cipher` package, so it could not be executed outside the
project regardless, and its internal symbols are not part of what this bundle
discloses. The method below is complete enough to judge the numbers.

## Setup

- **Synthetic vaults** generated by `gen_synthetic_vault.py` (published, standalone,
  seeded) at sizes **{19, 100, 500, 1000}** notes. Same schema, folder layout, and
  filename patterns as the real vault; folder mix roughly matches the real vault
  (conversations-heavy); every file flagged `synthetic: true`.
- Each vault is indexed once by the system's normal indexing path (BM25 + dense
  embeddings via a local `nomic-embed-text` on Ollama). Index-build time is recorded.
- **Machine:** Apple M3 MacBook Air (fanless), 16 GB, macOS. Single run, 2026-06-07.

## What is measured (the existing, unmodified code paths — no parallel fast path)

1. **Lens render** — the eight vault-backed Brain View endpoints, each driven through
   the app's real HTTP path via FastAPI's `TestClient` (`GET /brain/{timeline,
   commitments, person/…, project/…, graph, review, trash, growth}`). The ninth lens
   (Tools) renders from MCP configuration, not the vault, and is excluded.
2. **Recall** — `recall()` on two paths: a temporal/date-range query and a
   person-scoped query.

## Timing protocol

- `time.perf_counter()` around each call.
- **One warm-up call per metric, discarded** (recorded separately as the `cold_ms`
  figure in the results).
- Then **N = 30** timed calls; report **p50** (median) and **p95** (the
  `ceil(0.95·N)`-th sample).
- Per-machine runtime state (staged-write/trash/slot/MCP-audit/MCP-config stores) is
  redirected to a throwaway working directory so nothing touches the real vault or
  the machine's `.config` — the same isolation the test suite uses.

## Output

One JSON object per `(size, metric)` appended to
`results/scale_benchmark_2026-06-07.jsonl`, plus a header line recording seed, sizes,
N, machine, and the `synthetic: true` flag. Every row is verbatim runner output.

## Honesty note

All numbers here are from **synthetic** vaults and are labeled as such throughout the
paper; live-usage figures (e.g. the ~4.5 ms lens render, 181 ms cold recall) come from
the real ~19-note vault and are reported separately.
